## Deploy in 5 Minutes

### Step 1: Push Migration to Supabase
```bash
npx supabase db push
```

### Step 2: Create Storage Bucket (Manual)
1. Go to Supabase Dashboard → Storage
2. Click "Create a new bucket"
3. Name: `website-assets`
4. Toggle to Public
5. Click "Create bucket"

### Step 3: Hard Refresh Browser
```
Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
```

### Step 4: Verify It's Fixed
Open DevTools Console → Should NOT see these errors:
- ❌ `Multiple GoTrueClient instances detected`
- ❌ `Storage bucket 'website-assets' does not exist`

## Still Having Issues?

If errors persist after deployment:

1. **Clear everything**:
   - DevTools → Application → Clear storage
   - Restart browser
   - Hard refresh page

2. **Check migration applied**:
   - Supabase Dashboard → SQL Editor
   - Run: `SELECT * FROM storage.buckets WHERE name = 'website-assets';`
   - Should return one row

3. **Check imports consolidated**:
   - Search codebase for `from '../supabaseClient'`
   - Should return 0 results (use `../lib/supabase` instead)
